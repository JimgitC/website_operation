# 编写Web可访问性
#### 摘要

此页面介绍了一些基本注意事项，以帮助您开始编写残障人士更容易访问的Web内容。这些技巧是帮助您满足Web Content Accessibility Guidelines（WCAG）要求的良好实践。遵循相关WCAG要求的链接，“理解”文档中的详细背景，教程指南，用户案例等。

#### 页面内容
   + 提供内容丰富，独特的页面标题
   + 使用标题传达含义和结构
   + 使链接文字有意义
   + 为图像编写有意义的文本替代
   + 创建多媒体文字记录和字幕
   + 提供明确的指示
   + 保持内容简洁明了
   
### 提供内容丰富，独特的页面标题

对于每个网页，请提供简短的标题，以描述页面内容并将其与其他页面区分开。页面标题通常与页面的主要标题相同。将独特且最相关的信息放在首位；例如，将页面名称放在组织名称之前。对于属于多步骤过程的页面，请在页面标题中包括当前步骤。

### 使链接文字有意义

编写链接文本，以描述链接目标的内容。避免使用歧义的链接文本，例如“单击此处”或“阅读更多”。指示有关链接目标的相关信息，例如文档类型和大小，例如“建议文档（RTF，20MB）”。

### 创建多媒体文字记录和字幕

对于纯音频内容（例如播客），请提供成绩单。对于音频和视频内容（例如培训视频），也提供字幕。在成绩单和字幕中包括对理解内容很重要的语音信息和声音，例如“门吱吱作响”。对于视频成绩单，还包括重要视觉内容的描述，例如“ Athan离开房间”。

确保说明，指导和错误消息清晰，易于理解，并避免不必要的技术性语言。描述输入要求，例如日期格式。

### 保持内容简洁明了  
根据情况，使用简单的语言和格式。
   + 撰写简短明了的句子和段落。
   + 避免使用不必要的复杂单词和短语。考虑为读者可能不知道的术语提供词汇表。
   + 首次使用时请缩写。例如，Web内容可访问性指南（WCAG）。
   + 考虑为读者可能不知道的术语提供词汇表。
   + 适当使用列表格式。
   + 考虑使用图像，插图，视频，音频和符号来帮助阐明含义。

## 使链接文本更有意义
### 重点：

- 编写链接文本，以便描述链接目标的内容。避免使用含糊不清的链接文本，例如“连载动画”或“官方延申”。
![编写链接文本](https://images.gitee.com/uploads/images/2020/0325/222158_6cc02fe2_2228981.jpeg "bilibili-6.jpg")
- 用链接文本描述目标页面

- 指示有关链接目标的相关信息，例如分享代码和文件地址。
![相关信息](https://images.gitee.com/uploads/images/2020/0325/222305_f7061874_2228981.jpeg "bilibili-7.jpg")

## 为图像编写有意义的文本替代品
### 重点：能够提供图像信息或者功能的替代文本，对于美化装饰类的图像不需要替代文本

### 示例：用替代文本来传达图像功能或者信息
|无信息|信息化|
|:-----------|-----------:|
|一部于2018年6月开始每隔两个月更新一集的番剧。|一部于2018年6月开始每隔两个月更新一集的番剧。|
|**图片的替代文字**：“一部番剧”|**图像的替代文字**：“请于每两个月来看一次此番剧的更新”|

## 为多媒体创建成绩单和标题
### 重点：
- 应该向纯音频内容的播客提供成绩单。
- 音视频内容例如电影视频，还需要提供字幕。在成绩单和标题中包含对理解内容很重要的口头声音信息，例如“他来了”。
- 视频脚本还包括重要视觉内容的描述，例如“主播开始试吃螺狮粉”。

## 提供明确的说明
### 重点：

- 确保有利于理解的说明指导和错误消息通报，并避免使用不必要的技术语言。描述输入要求，例如日期格式。

## 保持内容简洁清晰
### 重点：
- 使用简短的段落或者句子来说明上下文
- 使用列表格式使整体阅读效率提高
- 避免术语或者复杂性词汇的出现，如有出现要提供可查询的词汇表（首次出现的词汇缩写要进行说明）
- 可使用音视频或图像或特殊符号来进行意义的阐述说明
